1) В скрипте есть 2 варианта реализации распределения вероятности отправки запросов. В папке UC_TZ используется плагин bzm - Weighted Switch Controller. В папке UC_TZ(ifContf) используется If controller. 
2) Пейсинг в обоих случаях используется одинаковый.
3) В скрипте присутствуют 2 тред группы, одна стандартная использовалась для отладки, вторая - "jp@gc - Ultimate Thread Group", скачена с помощью менеджера плагинов.
4) Вместе со скриптом лежат 2 файла, в файле "usernames" тестовый логин и пароль для авторизации (стоит увеличить количество учёток, чтобы их было как минимум не меньше чем вузеров в тесте, а в идеале сильно больше). 
5) В файле "registerData" список уникальных тестовых учёток для регистраций (количество уникальных пользователей рассчитано на 1 час теста при 15 rps, предполагаю откат БД к точке, созданной перед тестом, чтобы удалить созданных пользователей и использовать этот же пулл в следующем тесте, поэтому не генерирую учётки в скрипте, заодно не трачу ресурсы на генерацию.). 

Профиль рассчитан исходя из того что получение токена гостя происходит каждую итерацию, для своего удобства ушёл от rps к респонсам в час.

310% = 18 000 запросов в час
100% = 5806 - рассчитал количество итераций
